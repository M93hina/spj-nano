# B1Fフリースペース混雑度推定・予測システム 事前分析レポート

対象データ: `data/sensor_data.db` sensor_readings(2026-06-01 00:00〜2026-07-01 12:06, Asia/Tokyo換算、5分間隔、7センサー、各約8,780件)

## 1. 基礎統計

- 全センサーでCO2・温度・湿度の欠測はゼロ。計測間隔は中央値300秒で非常に安定、15分超のギャップは401(1件のみ、15分)。データ品質は良好。
- B1F EH: CO2 391〜2797ppm、mean 550.1±109.4。最大2797ppm(6/29 11:55)と2443ppm(同日15:55)は、直前まで滑らかに上昇していたトレンドから5分でΔ2000ppm超跳ね上がり、数ステップで元の傾向線に戻るという不自然な単発〜2点スパイク。**同時刻の他6センサーは全て平常値**であり建物全体の事象ではない。B1F EH固有のローカル事象かセンサーノイズと判断し、運用時は変化率フィルタでの除去を推奨(件数は0.02%と軽微)。
- 301/401/403は900〜1100ppm台まで滑らかに上昇しており、こちらは実利用による正常な変動。

## 2. B1F EH 無人時ベースライン

- 深夜1-5時(全日 n=1440): median=497.0(平日500.0 / 休日492.0、差は僅か)
- 日次の深夜中央値: 30日間で平均497.6、標準偏差45.0(範囲401.5〜583.0)→ 日々の変動は±9%程度で、実用上「安定して取得可能」
- 休日終日: median=468.0、90%点543
- **重要な発見**: 深夜(median約500)より、平日朝8-10時(median 458〜472)・休日全体の方がむしろCO2が低く安定している。深夜は換気低下により前日残留CO2が抜けきらないためと推測される。
- 結論: **B1F EH無人時ベースライン ≈ 460〜470ppm(変動幅±30ppm)**。深夜帯を基準にする場合は+30ppm程度のオフセットを考慮すべき。

## 3. データ駆動の混雑レベル閾値(提案)

平日日中9-18時の分位点: 5%=444, 10%=459, 25%=504, 50%=624, 70%=669, 75%=681, 90%=752, 95%=782, 99%=839(mean 609.3±127.5)

**B1F EH専用4段階閾値の提案**:

| レベル | CO2範囲 |
|---|---|
| 1: 空席・快適 | 〜550ppm |
| 2: やや利用あり | 550〜650ppm |
| 3: 混雑 | 650〜760ppm |
| 4: 非常に混雑 | 760ppm〜 |

現行固定閾値(500/700/1000)との比較: 500ppmは概ね妥当だが、700ppmは日中分布の80〜90%点相当で既にかなり混雑した状態、1000ppmは観測1ヶ月間で通常利用時には**一度も到達していない**(異常値2点を除く最大は901ppm)。固定閾値はB1F EHの実際のダイナミックレンジ(約450〜850ppm)に対して過大で、再キャリブレーションが必要。

## 4. 他階センサーとの相関

- 1F/3F/4F EH同士は相関0.68〜0.74と高く、共通の空調・外気変動が支配的。
- B1F EHとの相関は中程度(1F: 0.22, 3F: 0.53, 4F: 0.35、全体)。日中は3F(0.64)・4F(0.54)まで上がるが1Fは0.33止まり。深夜は逆にB1F-1F(0.38)・B1F-3F(0.41)が接近。→ **深夜は共通変動(空調・外気)が支配的で相関が揃い、日中はB1F固有(人由来)変動が乗って他階との相関が崩れる**という構造が見え、両成分の分離に見込みあり。
- 休日はB1F-1F相関が0.48まで上昇(平日0.17)、人の出入りが減ると共通変動が見えやすくなる傾向も確認。
- センサー間オフセット(深夜1-5時median): B1F EH=497, 1F EH=473, 3F EH=464, 4F EH=465, 301=443, 401=449, 403=430。B1F EHは無人時でも他センサーより25〜65ppm高いオフセットを持つ(地階特有の換気特性と推測)。モデル化時はこのオフセット補正が必要。

## 5. 時間パターン(曜日×時刻)の再現性

- 曜日×5分ビンの過去平均(1曜日あたり平均4.28週分)で実測値を説明: 全体R²相当0.845(残差std40.8、CO2std103.6)、平日日中R²相当0.828(残差std44.8、CO2std107.8)、深夜R²相当0.604。
- **曜日×時刻の単純な過去平均だけでCO2変動の8割以上を説明可能**であり、混雑予測の強力なベースライン特徴量になる。
- 週間ばらつきは利用が活発な時間帯(12-17時、20-22時)で45〜61ppmとやや大きく、日々のイベント性変動の考慮が別途必要。
- 曜日別の日中平均: 月611.6/火633.8/水568.4/木607.5/金602.6/土451.4/日459.7と平日間にも差あり。水曜日の週次推移(6/3=496.5→6/10=576.9→6/17=596.8→6/24=603.3)のように緩やかな上昇トレンドも観測され、定常季節性だけでなくトレンド成分の考慮も要検討。

## 6. 温度・湿度の活用可能性

- CO2 vs 温度: 全体-0.126(季節交絡)、平日日中限定+0.397(在室による発熱・空調負荷)。短周期偏差同士でも+0.197。
- CO2 vs 湿度: 全体-0.238、日中-0.034とほぼ無相関。CO2上位10%時と下位10%時で湿度平均はほぼ同一(61.5 vs 63.6)。
- 結論: **温度は弱い補助シグナル(相関0.4程度)、湿度はほぼ有用な追加情報にならない**。

## 総合結論(成立性判定)

**「B1F特化の混雑度推定・予測システム」構築アプローチは成立性が高い**と判断。

- データ品質は良好(欠測なし、間隔安定)。
- 無人時ベースラインは実用上安定して抽出可能(約460〜470ppm、日次変動±9%程度)。
- データ駆動閾値(550/650/760ppm)は現行固定閾値(500/700/1000)より実態に即しており、再キャリブレーションを推奨。
- 他階との相関から共通変動(空調・外気)とB1F固有変動(人由来)の分離に見込みがある(要センサーオフセット補正)。
- 曜日×時刻ベースラインだけでCO2変動の8割超を説明でき、強力な予測特徴量になる。
- 温度は補助シグナル、湿度はほぼ無効。
